# Git 提交说明与分支命名规范

本文档用于规范项目中的 Git 提交信息和分支命名，方便后续查看历史记录、协作开发和维护项目。

---

## 1. 常用提交类型

| 类型         | 含义                 | 示例                        |
| ---------- | ------------------ | ------------------------- |
| `feat`     | 新增功能               | `feat: 添加数据中心页面`          |
| `fix`      | 修复问题               | `fix: 修复球队切换后数据不更新的问题`    |
| `docs`     | 修改文档               | `docs: 更新 README 使用说明`    |
| `style`    | 样式修改，不影响功能逻辑       | `style: 优化首页卡片布局`         |
| `refactor` | 代码重构，不新增功能也不修复 bug | `refactor: 重构数据加载逻辑`      |
| `test`     | 测试相关修改             | `test: 添加数据处理测试`          |
| `chore`    | 配置、依赖、杂项维护         | `chore: 更新 .gitignore 文件` |
| `perf`     | 性能优化               | `perf: 优化数据加载速度`          |

---

## 2. 提交更改建议

建议遵守以下原则：

1. 一次提交只做一类事情；
2. 提交说明要具体，不要只写“修改”或“更新”；
3. 提交前检查是否误提交 `.env`、密钥、缓存文件、临时文件；
4. 如果修改内容较多，可以拆成多个 commit；
5. 提交说明尽量使用统一格式，方便后续维护。

---



## 3. 常用分支类型

| 前缀          | 用途       | 示例                       |
| ----------- | -------- | ------------------------ |
| `feature/`  | 新功能开发    | `feature/add-ai-report`  |
| `fix/`      | 修复 bug   | `fix/team-data-not-sync` |
| `style/`    | 页面样式调整   | `style/home-card-layout` |
| `docs/`     | 文档修改     | `docs/update-readme`     |
| `refactor/` | 代码重构     | `refactor/api-client`    |
| `test/`     | 测试相关     | `test/add-data-tests`    |
| `chore/`    | 配置、依赖、杂项 | `chore/update-gitignore` |

---

## 4. 分支命名规则

建议统一遵守：

1. 使用英文小写；
2. 单词之间用 `-` 连接；
3. 类型和任务之间用 `/` 分隔；
4. 不使用空格；
5. 尽量不使用中文；
6. 名字不要太长，但要能看出分支目的。
