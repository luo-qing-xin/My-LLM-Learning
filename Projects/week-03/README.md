# Week 03-04: API 调用大模型与 Benchmark 评测

本阶段的主要目标是学习如何通过 API 调用现有大模型，并进一步掌握如何使用 API 对大模型进行 benchmark 评测。  
本次任务预计用两周完成，本周交流时不要求全部完成，但需要汇报当前进展、已完成内容、遇到的问题和下一步计划。

---

## 一、Goals

通过本阶段的学习，需要完成以下目标：

1. 掌握使用 API 调用大模型的基本流程；
2. 理解 API 调用中常见参数的含义与作用；
3. 学会通过设计和修改 Prompt，让大模型完成鸢尾花分类任务；
4. 对比 GPT-2 与当前先进大语言模型在分类任务上的表现差异；
5. 掌握如何调用 API 对现有大模型进行 benchmark 评测；
6. 学会配置已有 benchmark 评测代码环境；
7. 掌握多线程或异步方式调用 API；
8. 能够将模型推理结果保存为 `json` 或 `jsonl` 格式；

## 二、Task List

### 1. 探索 API 调用大模型的基本方法

**任务内容：**

学习如何使用 API 的方式调用现有大模型，并理解 API 调用时各个参数的作用。

**参考资料：**

* Prompt Engineering Guide：https://prompt-guide.xiniushu.com/
* Prompt Engineering 中文资料：https://prompt-engineering.xiniushu.com/
* OpenAI API 中文资料：https://openai.xiniushu.com/

**重点关注：**

* API key 如何配置；
* base_url 的作用；
* model 参数如何选择；
* messages / prompt 的区别；
* system、user、assistant 三种角色的含义；
* temperature 的作用；
* top_p 的作用；
* max_tokens 的作用；
* stream 的作用；
* presence_penalty 的作用；
* frequency_penalty 的作用；
* stop 参数的作用。

**产出要求：**

对每一个重要参数，至少展示 3 个 case，用来说明该参数的作用。

每个 case 需要包含：

* prompt 输入；
* 参数设置；
* 模型输出；
* 对输出结果的简单分析。

**可记录内容示例：**

```text
参数：temperature

Case 1:
temperature = 0.1
Prompt:
请用一句话介绍人工智能。

模型输出：
......

观察：
输出较稳定，表达较保守。

Case 2:
temperature = 0.7
Prompt:
请用一句话介绍人工智能。

模型输出：
......

观察：
输出更加自然，有一定变化。

Case 3:
temperature = 1.2
Prompt:
请用一句话介绍人工智能。

模型输出：
......

观察：
输出更发散，但可能出现不稳定或不准确内容。
```

### 2. 设计 Prompt 完成鸢尾花分类任务

**任务内容：**

在鸢尾花数据集上设计 Prompt，让大模型完成分类任务，并通过修改 Prompt 格式观察模型表现变化。

**数据集：**

```python
from sklearn.datasets import load_iris
iris = load_iris()
```

**重点关注：**

* 如何把鸢尾花样本特征写入 prompt；
* 是否提供类别说明会影响结果；
* 是否提供 few-shot 示例会影响结果；
* 是否要求模型只输出标签会影响结果；
* prompt 的格式是否会影响准确率；
* 模型是否真的理解特征与类别之间的关系，还是只是在猜测。

**建议尝试的 Prompt 格式：**

1. 零样本 Prompt；
2. 带类别说明的 Prompt；
3. Few-shot Prompt；
4. 表格形式 Prompt；
5. JSON 形式 Prompt；
6. 只允许输出类别编号或类别名称的 Prompt。

**产出要求：**

需要从两个角度展示 Prompt 修改带来的区别：

1. **整体准确率对比**  
   比较不同 Prompt 格式下的分类准确率。

2. **具体 case 展示**  
   展示若干具体样本的输入、模型输出和真实标签，并分析模型为什么答对或答错。

**可记录内容示例：**

```text
Prompt 版本 A：直接提问
准确率：......

Prompt 版本 B：加入类别说明
准确率：......

Prompt 版本 C：加入 few-shot 示例
准确率：......

观察：
- 加入类别说明后，模型输出更加稳定；
- few-shot 示例可能提升准确率，但也可能让模型过度模仿示例；
- 要求模型只输出标签，可以减少无关文本，方便自动评测。
```

### 3. 使用 GPT-2 模型分类鸢尾花，并对比当前先进大模型

**任务内容：**

尝试使用 GPT-2 完成鸢尾花分类任务，并将其表现与当前先进大语言模型进行对比。

**重点问题：**

* GPT-2 能不能完成鸢尾花分类？
* 如果能完成，它是如何完成的？
* 如果表现不好，原因是什么？
* 它和当前先进 LLM 的差距在哪里？
* 是模型能力不足，还是 Prompt 设计不合适？
* 是否可以通过 few-shot、格式约束、后处理等方式提升表现？
* 这个任务是否适合用 GPT-2 来做？

**对比维度：**

* 准确率；
* 输出格式稳定性；
* 对数值特征的理解能力；
* 对类别标签的遵循能力；
* 是否容易产生无关解释；
* 是否容易输出非法类别；
* 推理成本；
* 使用难度。

**产出要求：**

完成一份对比分析，不能只写准确率，还需要解释：

* GPT-2 哪里做得好；
* GPT-2 哪里做得不好；
* 当前先进 LLM 为什么可能表现更好；
* 如果要提升 GPT-2 的表现，可以尝试哪些方法；
* 哪些问题可能是 GPT-2 无法通过简单 Prompt 解决的。
